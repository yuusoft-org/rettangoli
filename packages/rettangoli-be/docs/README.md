# rettangoli-be docs

Consolidated docs for **application code design**.

## Read In This Order

1. `framework-constitution.md` (goals and decision rules)
2. `framework-dream-state.md` (best possible end-state)
3. `application-specification.md` (normative app files, YAML, tests, conventions)
4. `application-specification-gap-analysis.md` (current spec vs ideal target)

These docs describe user-application structure and conventions.
They do not describe internal framework source layout.
