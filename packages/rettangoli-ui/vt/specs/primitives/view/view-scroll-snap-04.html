--- title: View - Scroll Snap - Responsive Landing Strip description: Verifies a landing-page strip can peek on desktop and collapse to full-width slides on smaller viewports. viewport: - id: desktop width: 1400 height: 760 - id: mobile width: 420 height: 840 specs: - 'Desktop uses wider partial-width slides to preview the next item.' - 'Smaller viewports promote slides to `1/1fb` so each panel reads as a full card.' - 'The same primitive structure scales across breakpoints.' --- Responsive marketing strip On desktop the layout peeks the next card; on smaller screens the cards become full-width slides. Built from `rtgl-view` plus native scroll snap. No carousel runtime is needed for this first version. The browser owns the scrolling; Rettangoli owns the composition. Faster iteration Keep the public API small until the pattern hardens. This is usually the right tradeoff for landing pages and editorial surfaces. Better defaults Use center snapping when the next card should remain visible. Use start snapping when the slide should feel like a full hero panel. Low maintenance Later, formalize only the CSS you keep reusing. That usually means `snap`, `snap-align`, and maybe `snap-stop` attrs on the primitive layer.
